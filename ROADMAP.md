# Roadmap

Fiestaaa is maintained as a focused private-event planning application. The
public repository accepts contributions that improve the official API,
developer experience, reliability, security, documentation, and test coverage.

## Good Fits

- Event, invitation, carpool, item-list, poll, shared-expense, QR, profile,
  notification, and admin payment-provider improvements that match the current
  product direction.
- API correctness fixes, permission hardening, privacy improvements, and
  better validation.
- Migrations, tests, CI hardening, dependency maintenance, and public
  documentation.
- Small refactors that reduce complexity without changing product behavior.

## Discuss First

- New third-party integrations.
- Large authentication, permission, encryption, privacy, or notification
  changes.
- New public API routes or response-shape changes.
- Database migrations that need backfills or production rollout coordination.
- Features that require frontend changes or production operations work.

## Out of Scope

- Official support for unrelated self-hosted deployments.
- Production deployment, backup, monitoring, incident-response, and rollback
  runbooks.
- Production secrets, service accounts, signing material, or infrastructure
  inventory.
- Changes that reuse Fiestaaa branding for public forks or derived services.

## Maintainer Notes

Release timing, versioning, production deployment, migration rollout, rollback,
and final scope decisions remain maintainer responsibilities.
