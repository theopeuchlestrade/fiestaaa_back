# Governance

Fiestaaa uses lightweight maintainer-led governance.

## Maintainer

The current maintainer is `@theopeuchlestrade`.

The maintainer is responsible for:

- repository direction and scope;
- accepting or rejecting pull requests;
- release decisions;
- security response;
- dependency and infrastructure policy;
- final decisions when consensus is not reached.

## Contribution Model

Contributions are welcome when they fit the project scope and quality bar.

For non-trivial work, open an issue or draft pull request first. This avoids
large changes being built against assumptions that may not match the project
direction.

## Decision Making

Technical decisions are made through discussion in issues and pull requests.
The preferred outcome is rough consensus backed by clear reasoning, tests, and
maintainable implementation.

When tradeoffs remain, the maintainer makes the final call.

## Releases

Releases are cut by the maintainer through the repository release workflow.
Versioning, changelog updates, deployment timing, and rollback decisions remain
maintainer responsibilities.

## Becoming a Maintainer

Additional maintainers may be added after sustained, high-quality
contributions and demonstrated care for project maintenance, security, and
review quality. There is no automatic promotion path.

## Policy Changes

Governance, support, conduct, security, and brand policies may evolve as the
project grows. Policy changes should be reviewed like other project changes.
