# Brand and Asset Policy

This repository's source code is licensed under `AGPL-3.0-only`. That license
applies to the covered source code. It does not grant a general right to use
the Fiestaaa name, logos, app icons, domain names, screenshots, product copy,
or other brand assets.

This policy is intended to make the project easier to fork and contribute to
without creating confusion about what is official.

## Allowed Use

You may use the Fiestaaa name and repository references to:

- describe this project accurately;
- link to the original repository;
- say that a project is a fork of, derived from, or compatible with Fiestaaa;
- include screenshots or logs when reporting bugs, submitting pull requests, or
  documenting development work.

These uses must not imply endorsement, sponsorship, or official status unless
that has been explicitly agreed.

## Restricted Use

Without explicit permission, do not:

- use the Fiestaaa name, logo, app icon, or visual identity as the primary
  branding for a public fork, hosted service, package, app store listing, or
  commercial offering;
- register confusingly similar domains, social accounts, package names, or
  application identifiers;
- present a modified version as the official Fiestaaa project;
- use Fiestaaa brand assets in a way that suggests endorsement or partnership.

## Forks and Deployments

Public forks and independently deployed versions should use their own name,
icons, logos, visual identity, domains, and application identifiers. It is fine
to state factually that the project is based on Fiestaaa, as long as users can
clearly tell it is not the official project.

## Third-Party Assets

Third-party names, logos, and service marks remain the property of their
respective owners. Their presence in this project, if any, does not grant extra
rights to reuse them outside the purpose for which they appear.

## Questions

For permission requests or unclear cases, contact the maintainer through the
project's normal private channel.
